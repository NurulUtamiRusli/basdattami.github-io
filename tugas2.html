MySQL adalah sebuah perangkat lunak sistem manajemen basis data SQL (bahasa Inggris: database management system) atau DBMS yang multialur, multipengguna, dengan sekitar 6 juta instalasi di seluruh dunia. MySQL AB membuat MySQL tersedia sebagai perangkat lunak gratis di bawah lisensi GNU General Public License (GPL), tetapi mereka juga menjual di bawah lisensi komersial untuk kasus-kasus di mana penggunaannya tidak cocok dengan penggunaan GPL.

MySQL adalah sebuah implementasi dari sistem manajemen basis data relasional (RDBMS) yang didistribusikan secara gratis di bawah lisensi GPL (General Public License). Setiap pengguna dapat secara bebas menggunakan MySQL, tetapi dengan batasan perangkat lunak tersebut tidak boleh dijadikan produk turunan yang bersifat komersial. MySQL sebenarnya merupakan turunan salah satu konsep utama dalam basisdata yang telah ada sebelumnya; SQL (Structured Query Language). SQL adalah sebuah konsep pengoperasian basisdata, terutama untuk pemilihan atau seleksi dan pemasukan data, yang memungkinkan pengoperasian data dikerjakan dengan mudah secara otomatis.

Kehandalan suatu sistem basisdata (DBMS) dapat diketahui dari cara kerja pengoptimasi-nya dalam melakukan proses perintah-perintah SQL yang dibuat oleh pengguna maupun program-program aplikasi yang memanfaatkannya. Sebagai peladen basis data, MySQL mendukung operasi basisdata transaksional maupun operasi basisdata non-transaksional. Pada modus operasi non-transaksional, MySQL dapat dikatakan unggul dalam hal unjuk kerja dibandingkan perangkat lunak peladen basisdata kompetitor lainnya. Namun pada modus non-transaksional tidak ada jaminan atas reliabilitas terhadap data yang tersimpan, karenanya modus non-transaksional hanya cocok untuk jenis aplikasi yang tidak membutuhkan reliabilitas data seperti aplikasi blogging berbasis web (wordpress), CMS, dan sejenisnya. Untuk kebutuhan sistem yang ditujukan untuk bisnis sangat disarankan untuk menggunakan modus basisdata transaksional, hanya saja sebagai konsekuensinya unjuk kerja MySQL pada modus transaksional tidak secepat unjuk kerja pada modus non-transaksional.

contoh penerapan MySQL



keterangan
1. Id, menandakan nomor urut select query seperti pada mahasiswa memiliki id 1 dan penerima_beasiswa memiliki id 2.
2. Select_type, menandakan menunjukkan tipe dari statement yang digunakan, seperti contoh pada tabel mahasiswa menggunakan primary sehingga akan melakukan pencarian yang dimulai dari bagian terluar query utama, sedangkan pada tabel penerima_beasiswa menggunakan materialized sehingga hasil dari select suatu subquery akan disimpan dalam bentuk tabel sementara dan hanya dapat dieksekusi sekali.
3. Table, pada kolom table berarti query yang dimasukkan yaitu query dari colom table dan colom penerima_beasiswa
4. Type, bagaimana suatu tabel dapat tergabung, pada tabel mahasiswa melakukan pemindaian lengkap, sedangkan pada tabel penerima_beasiswa melakukan pemindaian oleh indeks suatu table.
5. Possible_keys, kunci yang dapat dipakai, pada tabel mahasiswa tidak membutuhkan kunci sedangkan pada tabel penerima_beasiswa membutuhkan primary key dalam menemukan mahasiswa yang tidak menerima beasiswa.
6. Key, kunci yang telah digunakan, dapat dilihat pada tabel mahasiswa tidak menggunakan kunci apapun dalam proses eksekusi sedangkan pada tabel penerima_beasiswa menggunakan kunci utama.
7. Key_len, panjang dari kunci yang digunakan.
8. Ref, kolom yang dibandingkan dengan index.
9. Rows, jumlah baris yang dicari, yang dimana kedua tabel tersebut memiliki 3 baris yang akan dieksekusi, sehingga banyaknya baris yang dieksekusi akan mempengaruhi lambat tidaknya efisiensi kerja.
10. Extra, seperti biasa extra merujuk pada informasi tambahan dari bagaimana menyelesaikan tiap query, yang dimana hanya pada tabel mahasiswa yang memiliki extra yakni ‘using where’ dalam menentukan kondisi baris tersebut dan pada tabel penerima_beasiswa memiliki extra ‘using index’ dalam menggunakan informasi dari pohon indeks tunggal tanpa perlu membaca ulang baris sebenarnya.
